## 🔐 登录方式

### 密码登录
1. 选择"密码登录"模式
2. 输入测试账号的手机号
3. 输入对应的密码
4. 点击登录

**示例**:
```
手机号: 13800138000
密码: test123456
```

### 验证码登录
1. 选择"验证码登录"模式
2. 输入测试账号的手机号
3. 点击"发送验证码"（实际不会发送，只是模拟）
4. 输入该账号对应的专属验证码
5. 点击登录

**示例**:
```
手机号: 13800138000
验证码: 888888
```

---

## ⚠️ 重要提示

1. **非测试账号无法登录**: 其他手机号会直接拒绝登录
2. **密码和验证码必须正确**: 即使是测试账号，密码或验证码错误也会登录失败
3. **验证码不会真实发送**: 点击"发送验证码"只是模拟操作，请直接使用上述专属验证码

## 🧪 测试场景

### 测试登录成功
- 使用正确的测试账号 + 正确的密码/验证码
- **预期**: 登录成功，跳转到首页

### 测试登录失败 - 密码错误
- 使用测试账号 + 错误的密码
- **预期**: 提示"密码错误，请检查后重试"

### 测试登录失败 - 验证码错误
- 使用测试账号 + 错误的验证码
- **预期**: 提示"验证码错误，请检查后重试"

### 测试登录失败 - 非测试账号
- 使用非测试账号的手机号（如: 13912345678）
- **预期**: 提示"该手机号不是测试账号，请使用正确的测试账号登录"

## 📝 修改测试账号

如需添加或修改测试账号，请编辑文件：
```
src/features/AuthModule/config/testAccounts.ts
```

在 `TEST_ACCOUNTS` 数组中添加新的账号配置：

```typescript
{
  phone: '13800138003',           // 手机号
  password: 'your_password',      // 密码
  smsCode: '111111',              // 专属验证码
  nickname: '测试用户C',           // 昵称
  avatar: 'avatar_url',           // 头像（可选）
  verified: true,                 // 是否已认证
  description: '账号说明',         // 说明（可选）
}
```
